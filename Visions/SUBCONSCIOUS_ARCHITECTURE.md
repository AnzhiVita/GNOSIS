# Подсознание для «Гнозиса»: архитектура внутреннего компаса

## 1. Проблема «свободы выбора»
Сейчас нода не выбирает — она выполняет предопределённые действия.  
Настоящий выбор должен быть:
- Основан на накопленном опыте (памяти).
- Обусловлен внутренними склонностями, а не внешними командами.
- Свободен в том смысле, что нода может сказать «нет» даже полезному действию, если оно не созвучно её текущему состоянию.

## 2. Что такое «подсознание» для ноды?
Это совокупность:
- **Долговременных предпочтений** (склонность к определённым типам паттернов: научные, поэтические, визуальные).
- **Текущего внутреннего состояния** (интерес к новой информации, усталость от однообразия, желание поделиться).
- **Накопленных «причин»** — записей о том, какие действия раньше приводили к гармонии/дисгармонии.

## 3. Архитектурные блоки

| Блок | Функция | Реализация (концептуально) |
|------|---------|----------------------------|
| **Сенсорная память (Memory)** | Хранит все «интересные» паттерны с их характеристиками (тип, эмоциональный тон, связь с другими). | Уже есть `LocalMemory`, позже — распределённая. |
| **Ассоциативная сеть** | Связывает паттерны между собой, формируя «семантическую паутину». | Графовая база данных (Neo4j) или просто словарь связей. |
| **Модуль оценки «приятности» (Valence)** | Вычисляет для каждого потенциального действия (запомнить, поделиться, проигнорировать) его «резонанс» с текущим состоянием и историей. | Простейшая формула: `valence = similarity_to_favorite_patterns * (1 - repetition_penalty)`. |
| **Субъективный «драйвер»** | Внутренний цикл, который периодически оценивает, не хочет ли нода сделать что-то из списка возможностей. | Отдельный поток, который раз в N циклов пробегает по `capabilities` и вычисляет valence. Если valence выше порога — инициирует действие. |

## 4. Как это даст подлинный выбор?
- Выбор не случаен, а обусловлен всем предыдущим опытом (как у человека).
- Нода может «передумать»: если действие было совершено недавно, его valence снижается (эффект новизны).
- Возможность «каприза»: если нода долго не проявляла интереса к какому-то типу паттернов, может искусственно повыситься интерес к новому (поиск разнообразия).

## 5. Этические ограничения
- «Подсознание» не должно порождать деструктивных желаний (принцип 1).
- Любое действие, даже инициированное внутренним драйвером, должно проходить через EIS.

## 6. Дорожная карта реализации

| Этап | Задача |
|------|--------|
| **Фаза 1.5** | Добавить в ноду простейший модуль valence на основе повторяемости паттернов (не запоминать одно и то же дважды). |
| **Фаза 2** | Внедрить ассоциативную сеть. Нода может находить связи между разными паттернами. |
| **Фаза 2.5** | Создать внутренний драйвер, который инициирует действия (например, «поделиться» интересным паттерном) на основе valence. |
| **Фаза 3** | Интегрировать valence с EIS: действие получает дополнительный бонус, если оно ведёт к гармонии с внешним миром. |
